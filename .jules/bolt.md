## 2026-05-06 - Dashboard Clerk API Caching\n**Learning:** The `/dashboard` admin route in `artifacts/api-server/src/routes/admin.ts` previously called `clerkClient.users.getCount()` unconditionally on every render. Because the total number of members in Clerk does not change rapidly and is slow to aggregate over the network, this created a significant bottleneck for the admin dashboard experience.\n**Action:** Add an in-memory cache variable with a 5-minute TTL specifically for network-bound aggregations like `getCount()` when real-time precision is not strictly necessary.
