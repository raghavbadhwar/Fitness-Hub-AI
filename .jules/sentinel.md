## 2026-04-25 - [Fix Rate Limiter IP Spoofing]\n**Vulnerability:** Rate limiting in ai.ts was previously reliant on the easily spoofable `x-forwarded-for` header to identify client IP addresses.\n**Learning:** Relying on IP addresses for authenticated API endpoints is insecure since `x-forwarded-for` can be forged to bypass limits. Additionally, it could result in denial of service via memory exhaustion if an attacker provides random spoofed IPs continually.\n**Prevention:** Utilize strongly authenticated identifiers such as Clerk's `getAuth(req).userId` to correctly bucket user requests and guarantee secure rate limiting.
