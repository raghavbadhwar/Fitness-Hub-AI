## 2024-05-18 - Express IP Spoofing via X-Forwarded-For\n**Vulnerability:** The Clerk Proxy middleware extracted the client IP address by manually reading and splitting the `x-forwarded-for` header, enabling attackers to spoof their IP address by injecting a fake `X-Forwarded-For` header in their request. This bypasses downstream rate limits that rely on IP.\n**Learning:** Express has built-in mechanisms to securely parse client IPs through the `trust proxy` configuration. Manually parsing proxy headers like `x-forwarded-for` without validating trust boundaries introduces significant risk.\n**Prevention:** Always rely on Express `req.ip` for IP resolution, ensuring the application is correctly configured with `app.set("trust proxy", ...)` based on the deployment environment.
